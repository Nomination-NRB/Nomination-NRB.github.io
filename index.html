KMP

KMP算法是在BF算法上的提升：主串不进行回溯，子串回溯；子串回溯的位置由next数组来确定
Next数组：
这里我们引入一个新的概念：next数组，它的作用是保存子串 j 应该回溯的位置
即子串并不是每次匹配失败就回到最初的位置 j=0，而是尽可能的利用已经匹配成功的字符接着进行后续匹配
暴力匹配
S[i]!=T[j]，i就移动到下一位 b；j也就回溯到初始位置 a 与之重新尝试匹配

此操作效率比较低，因为我们没有利用子串中相同的前后缀

前后缀：
例如：求字符串 abcdab 的前后缀
| 前缀 | 后缀 |
| :—-: | :—-: |
| a | b |
| ab | ab |
| abc | dab |
| abcd | cdab |
| abcda | bcdab |
可以发现当 j 移动到d元素的时候，前面的字符串 abcdab 存在相同的前后缀（黄色方框），即ab，也就是d的前面有长度为2的最大相同前后缀，而这个值就是我们next数组所需要的值

接下来我们就来详细的求 T 字符串的next数组

  1. 当 j 指向 a 的时候，a 前面没有字符串，单独处理next[0] = -1（表示此时 j 与 i 不进行字符比较）
  2. 当 j 指向 b 的时候，b 前面有字符串 a ，但是它就一个字符，所以没有相同前后缀next[1] = 0
  3. 当 j 指向 c 的时候，c 前面有字符串 ab ，但是没有相同前后缀，next[2] = 0
  4. 当 j 指向 d 的时候，d 前面有字符串 abc ，但是没有相同前后缀，next[3] = 0
  5. 当 j 指向 a 的时候，a 前面有字符串 abcd ，但是没有相同前后缀，next[4] = 0
  6. 当 j 指向 b 的时候，b 前面有字符串 abcda ，有相同前后缀 a ，next[5] = 1
  7. 当 j 指向 d 的时候，d 前面有字符串 abcdab ，有相同前后缀 ab ，next[6]=2
    也就是说next[i]代表 i 前面的字符串的最大相同前后缀的值
    下标 0 1 2 3 4 5 6
    字符 a b c d a b d
    next[i] -1 0 0 0 0 1 2

BF算法

BF算法即字符串的模式匹配算法，又称为暴力匹配
我们将字符串分为主串与子串，从第一位开始校对，若比对失败，则子串回溯，主串移动到下一位
如下图：
​ 1.发现a与a匹配，主串与子串同时移动
​ 2.发现c与a不匹配，主串匹配的初始位置移动到下一位，即c，然后又不匹配，主串再移动一位
​ 3.发现，子串全部匹配成功，BF算法结束

方法：
​ 我们利用结构体来储存主串与子串，主要分为以下步骤
​ 1.串的初始化
​ 2.把字符数组赋给主串与子串
​ 3.打印串
​ 4.BF算法

节点结构

变量说明：
​ 1.data指针，相当于字符数组，用来存放字符串
​ 2.len，记录字符串的长度

字符串的初始化

具体操作：
​ 1.new一个s（字符串结构体指针，方便对它操作）
​ 2.初始化data为NULL，因为我们还不知道字符串的长度，不能在这里对data=new char[长度]
​ 3.初始化len为0

字符串的赋值

我们传入字符串x，来为s赋值
具体操作：
​ 1.我们要遍历字符数组来获得长度，所以需要设置工作指针temp
​ 2.如果len为0，则不对s进行操作
​ 3.如果len不为0，则将temp字符数组的数据拷贝给s
​ ///1.因为求长度len的时候，temp已经移动到字符数组的尾部了，所以需要先复位temp=x
​ ///2.更新s的数据域的len（s->len=len）
​ ///3.更新s的数据域的data，给它new一片空间，长度为len+1，因为字符数组的最后一位要保存**’\0’**
​ ///4.进行赋值

字符串的显示

具体操作：
​ 1.普通的遍历结构体数组

BF算法核心

两个字符串的匹配
具体操作：
​ 1.通过两个下标索引来移动主串与子串
​ 2.循环条件，只要主串与子串没有匹配完，就不退出循环
​ 若当前字符匹配成功，则索引 i 与 j 同时后移
​ 若子串匹配失败，则子串下标 j 回到第一个（j = 0），主串回到初始位置的下一位 （i = i - j + 1）
​ 例如：A与C不匹配，i从下标0移动到下标1，j回到下标0
​
​ 3.如果 j == len，代表子串的字符都匹配完成了，匹配成功
​ 否则匹配失败

循环队列

关于循环队列需明白以下几点：
1、循环队列是队列的顺序存储结构
2、循环队列用判断是否为空利用 Q.front==Q.rear
3、循环队列头指针始终指向队列头元素，而尾指针始终指向队列尾元素的下一个位置
4、按照队列的定义，队头删除，队尾插入

初始化队列

具体操作：
​ 1.new一个头节点，方便返回得到值
​ 2.初始化队头与队尾索引，同时赋值为0，表示队列为空

队空

**具体操作：** ​ 1.队列空：q->rear==q->front

队满

我们要牺牲队列里面的一个空间来标识队列满，首先front指向队头，rear指向队尾的下一个
如下图：刚开始front指向索引0，如果我们不牺牲一个空间的话，当队列满了，即填入数据5，rear索引指向5的下一个，即0，会发现front与rear相等，与队空的情况一样，无法区别队空与队满，所以索引5的位置是不能填入数据的（即牺牲这个空间），填入当填入数据4的时候，rear索引指向5

​ 1.队列满：(q->rear + 1) % MAXSIZE == q->front
​ (q->rear + 1) % MAXSIZE == q->front来实现队列循环的下标—越界问题
​
​ 当rear+1==front时就表示队列已满，但是索引回从5变到0
​ 为了让索引能够循环，我们利用取余运算，即（rear+1）%MAXSIZE，可以达到循环索引的目的
​ 例如，（5+1）%6=0，即回到front的索引

ps:遇到索引越界问题，我们大都采用取余的方法

入队

每次入队后，将队尾rear指向下一个位置
具体操作：
​ 1.入队需要判断队列是否已满
​ 2.将x入队，因为rear是指向队尾的下一个，所以直接用rear当数组的下标即可
​ 3.入队后rear索引要往后面移动一位，所以还是要注意越界问题**(q->rear + 1) % MAXSIZE**

出队

出队后，将front指向下一个位置
具体操作：
​ 1.出队需要先判断队列是否已空
​ 2.因为front也会指向下一个位置，所以仍然要注意越界问题q->front = (q->front + 1) % MAXSIZE

队列的显示

具体操作：
​ 1.队列的长度：队尾 - 队头 + MAXSIZE（加上MAXSIZE是为了防止出现负数的情况），再取余，不超过限制长度（ 队尾 - 队头 + MAXSIZE ) % MAXSIZE
​ 例如下图：rear-front=1-4=-3，-3+MAXSIZE=3，3%6=3，所以3才是队列的长度
​
​ 2.因为要遍历队列，所以我们要设置工作索引index，工作索引也会遇到越界问题，所以还是要**+1再取余**

链式队列

链式队列就是利用链表来实现队列这个数据结构
也可以这样理解：就是一个单链表，方法只有尾插法（入队），顺序删除（出队）

节点结构

我们依然利用结构体来实现队列（链表）
变量说明：
​ data：可自定义数据类型与个数
​ next：指向下一个结构体的指针

队列初始化

与链表初始化相同
具体操作：
​ 1.new一个头节点
​ 2.让头节点的next指针指向NULL（防止野指针的出现）
​ 3.返回头节点

队空

判断队列是否为空与链表是否为空一样
具体操作：
​ 1.若头节点的next指针为NULL，则队列为空

入队

链式队列的入队就类似链表的尾插法
我们需要遍历到队列（链表）的尾部，所以需要设置工作指针
具体操作：
​ 1.设置工作指针temp，并将工作指针移动到最后一个节点上
​ 2.new一个新节点，即要插入的节点，并将x赋给node的数据域
​ 3.目前temp->next是NULL，将它赋给新节点node的next（使node成为新的尾节点）
​ 4.再将temp->next赋值为node，让它们连接起来

出队

出队操作就类似链表的顺序删除
具体操作：
​ 1.先判断队列是否已为空，若为空则无法出队
​ 2.记录第一个节点temp（即head->next)
​ 3.将头节点的next指向第二个节点，因为我们要删除第一个节点（head->next = temp->next;），temp->next就是第二个节点
​ 4.删除第一个节点temp

队列的显示

队列的显示与链表的显示一样，因为要遍历队列，所以需要设置工作指针
具体操作：
​ 1.设置工作指针temp
​ 2.区别于链表的显示（在链表显示中，我们设置的工作指针是head，所以循环终止条件是temp->next!=NULL）
​ 在这里我们用head->next来当做工作指针，所以循环终止条件是temp!=NULL（其实两个方法都可以，根据个人喜好使用）

顺序栈(C++)

顺序栈，即栈的存储结构是利用一组地址连续的存储单元，依次存放自栈底到栈顶的数据元素，同时附设指针（索引）top指示栈顶元素
在顺序栈中的位置，top = 0表示空栈。由于栈在使用的过程中所需要的大小难以估计（不像链栈，链栈可以随时入栈，也就是加入新节点），所以通常是先为栈分配一个基本容量
使用的过程中，当栈的空间不够使用的时候再继续追加存储空间。我们以下述类型说明作为顺序栈的定义：

初始化栈

​ 我们利用C++的类来实现顺序栈（也可以用C语言的结构体）

变量说明：
​ 1.data：一个字符类型的数组
​ 2.size：顺序栈的实际大小
​ 3.top：充当栈顶的指针（并不是实际的指针*）（也可以叫做索引）
​ 4.MAX_SIZE：一个常量定义顺序栈的最大容量

入栈

由于栈的空间是有限制的，所以需要判断栈是否满
具体操作：
​ 1.判断栈是否已满
​ 2.栈顶索引top++（表示当前栈里面压入了一个元素）
​ 3.将元素压入，即数组data-[top]的值为ch

出栈

只有在栈不为空的时候才允许出栈操作
具体操作：
​ 1.栈顶索引top–
​ 2.返回栈顶元素（因为先自减了，所以要+1回到之前的栈顶位置）

出元素不出栈

与出栈类似，但栈顶索引不改变
具体操作：
​ 1.判断栈是否为空，若不为空则直接返回当前栈顶元素
​ 2.判断栈空实际上就是检查索引top是否为-1，或者调用函数isEmpty()

判断栈空

具体操作：
​ 1.判断索引top是否为-1，若等于-1，则为空，反之不为空

判断栈满

什么是链表？
链表是一种常见的重要的数据结构（本身不存在，我们利用现有的方法来实现它）它是动态地进行存储分配的一种结构；链表和数组比较，不用事先确定存储空间，而是根据需要开辟内存单元
补：什么是结构体？
我们可以把结构体理解为一个盒子，它可以装很多不同的数据；主要分为两大部分：1.数据域 2.指针域
